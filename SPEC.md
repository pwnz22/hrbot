## Task 5 — Приём отклика через бота (FSM)

**Цель:** кандидат заходит в бота (по deep-link из описания вакансии), бот по шагам собирает данные отклика и сохраняет в БД. Модераторы/админы получают уведомление.

### FSM `ApplyStates`
1. `waiting_name` — ФИО
2. `waiting_phone` — телефон
3. `waiting_email` — email
4. `waiting_message` — сопроводительное сообщение (или пропуск)
5. `waiting_resume` — файл резюме (pdf/docx) или пропуск
6. `waiting_confirm` — подтверждение и отправка

### Вход в FSM
- Deep-link: `/start apply_<vacancy_id>` — бот показывает карточку вакансии и кнопку «📨 Откликнуться».
- По нажатию кнопки — старт FSM.
- `/cancel` прерывает FSM.

### Модель данных
Таблица `applications` — добавить:
- `source` (`String(20)`, default `'gmail'`) — `'gmail'` или `'bot'`.
- `telegram_user_id` (`BigInteger`, nullable) — id кандидата в Telegram для связи.
- `gmail_message_id` — сделать **nullable** (сейчас `unique NOT NULL`, uniqueness остаётся).

### Хранение файлов
Резюме из бота сохраняется в `downloads/bot/<vacancy_id>/<application_id>_<filename>`.

### Уведомления
После сохранения — послать сообщение всем `TelegramUser` с ролью MODERATOR/ADMIN с кратким описанием отклика и кнопкой открыть его.

### Вакансии, доступные для отклика
- `deleted_at IS NULL` (задача 4 добавит `is_published`).
